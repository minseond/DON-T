
## **TIL - 260304**

**1. 잘한점**
- Java 21과 Spring Boot 3.5.10 등 최신 스택을 선택한 기술적 근거를 학습했다.
- 기획을 구체화 하기 위해 유저플로우를 제작하는 과정에서, 사용자 여정을 차트로 도식화하며 로직의 빈틈을 점검했다.

<br>

**2. 개선점**
- JDK 21과 Spring Boot 3.5.10 등 최신 스택을 도입하기로 결정했으나,
이 선택이 실제 핀테크 서비스 운영 환경에서 발생할 수 있는 대량 트래픽이나 동시성 이슈를 어떻게 해결해 줄지에 대한 구체적인 분석이 부족했다.
- 왜 이 기술 스택을 선택했는지 묻는 질문에 대해 비즈니스와 운영 관점에서의 답변을 준비하는 습관이 필요하다.
예상되는 문제 상황 시나리오를 미리 작성해 보는 시간을 갖자.

<br>

**3. 배운점**
- 핀테크 서비스의 백엔드 구성 시, 단순히 익숙한 버전을 사용하는 것이 아니라
프로젝트의 특성(고성능, 동시성 처리)에 맞는 최적의 스택을 선정하고 AI 기능을 유연하게 결합할 수 있는 구조를 고민해야 함을 알았다.
