### 작업 애로사항
**트러블 슈팅**
1. 포매터나 린트를 적용해서 하는 작업 중 pre-commit을 사용하는 작업은 처음이었음.
2. 오늘 13시부터 개발환경 설정에 대해서 고민하면서 코드를 작성하고 있었는데 중간 커밋단계에서 pre-commit을 하면서 오류가 발생했음.
3. 당연하게도 pre-commit이 잘못인줄 알고 1시간동안 이 코드 저 코드 바꿔가면서 너무 강하게 규칙을 잡은건가 싶었음.
4. 사실 그게 아니고 WSL문제였음. WSL을 사용하고 vscode에서 접속하는 경우 프로젝트 디렉토리를 mnt/c를 사용하는 경우가 있는데 오늘과 같은 경우였음.
5. git commit을 보낼때 mnt/c 디렉토리를 사용하면서 wsl을 사용하면 어디서 커밋을 보내는지 몰라 git이 오류를 내는 것 같다고 가정했음.
6. 그래서 프로젝트를 옮겨서 작업을 하는 방법과 새로 클론을 받는 방법, 그리고 wsl에서 일일히 git add와 git commit을 통해 cli에서 제어하는 방법을 생각해냄
7. 결론적으로 당장 실행해볼 수 있는 cli환경에서 직접 입력하는 방법을 써봤고 커밋을 올려본 결과 잘 올라가는 것을 확인할 수 있었음
8. 마지막으로 해당 프로젝트를 cli 커밋하고 난 이후에 wsl 의 폴더 ~/workspace에 직접 프로젝트를 클론받아 다시 적용해봄. 잘 올라가는 것을 확인할 수 있었음.
**개선사항**
- 다음에는 wsl을 사용할 때 mnt/c 디렉토리를 사용하지 않고 wsl의 홈 디렉토리에서 작업을 하도록 해야겠음.
- 실제로 mnt/c에서 작업하는 것보다 홈 디렉토리에서 사용하는 것이 성능도 더 좋다고함.

### 프로젝트 고려사항

또한 index를 자동으로 지정해주기도 하지만 직접지정하는 것이 좋을 때도 있음. 주의할 점은 index를 너무 많이 사용하거나 index로 사용할 수 없는 필드에 index를 지정해서 성능저하를 일으키지 않도록 주의해야함.
